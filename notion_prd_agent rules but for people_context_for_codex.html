[Hack092725] agent rules but for people

  • Coding agents like Claude and Codex have guidance files, e.g. Claude.MD
  • We need the same for humans so we don’t ask stupid questions and actually learn from using AI-assisted tasks like coding
  • Prototype: Learnings for both Agents and Humans
  • Augmented Human-AI Interaction

System Arch

Agent Roles

Knowledge Exchange agent

  • Retrieving
  • Reprompting
  • Synthesizing

Task Agent

  1. executes and interfaces with user (produces assistant message)
  1. accept new instructions (append to user message or assistant message)

User Flow

First pass: objective is to capture learning gaps of user and assistant

  1. <session>
  1. User asks what "UseEffect" is in React
  1. [Task Agent] Executes task
  1. [KE Agent] Logs entry of user request to Documents
  1. System creates log entry of "user actions"
  1. [KE Agent] Request information learned from user action + documents. Write to  Documents
  1. [KE Agent] Logs entry of Task Agent output into Documents
  1. </session>

Second pass: objective is to build faster with more knowledge

  1. Session Starts with learn mode toggled on
  1. User asks what "UseEffect" is in React FOR THE SECOND TIME
  1. [KE Agent] Retrieves previous user documents
  1. [KE Agent] Reprompting task based on previous user documents
  1. [Task Agent] Prompts human, "here's how we solved it last time, should we proceed"? < [producing some notion version of the markdown from step 3 >
  1. User approves "plan"
  1. [Task Agent] Executes task
  1. [KE Agent] Logs entry of user request to Documents
  1. System creates log entry of "user actions"
  1. [KE Agent] Request information learned from user action + documents. Write to  Documents
  1. [KE Agent] Logs entry of Task Agent output into Documents
  1. Session Ends

Implementation

OpenAI Agents SDK will handle session id for every session

Types of Documents:

1. user actions

2. synthesized learnings

3. agent actions
